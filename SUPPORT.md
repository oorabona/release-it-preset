# Support

This document explains how to get help with `@oorabona/release-it-preset`.

## Where to ask questions

Use [GitHub Discussions](https://github.com/oorabona/release-it-preset/discussions) for:
- "How do I set up X?"
- "Is this the right preset for my use case?"
- "I'm seeing unexpected behavior — could it be a misconfiguration?"
- Usage tips and sharing patterns

Discussions are the right venue for open-ended conversation and exploratory questions.

## Where to file bugs

Use [GitHub Issues](https://github.com/oorabona/release-it-preset/issues) with the **Bug Report** template for:
- Confirmed regressions (behavior that worked before and no longer does)
- CLI errors that don't match documented behavior
- Incorrect output from scripts or configs

Please search existing issues before filing a new one.

## Where to propose features

Use [GitHub Issues](https://github.com/oorabona/release-it-preset/issues) with the **Feature Request** template when you have a **clear, concrete use case** and a well-defined request.

For early-stage ideas or "is this a good fit?" debates, use [Discussions](https://github.com/oorabona/release-it-preset/discussions) instead.

## How to report security vulnerabilities

**Do NOT open a public issue for security vulnerabilities.**

Please follow the responsible disclosure process described in [SECURITY.md](SECURITY.md). You can report privately via the GitHub Security Advisory form or by emailing `olivier.orabona@gmail.com` directly.

## Response time expectations

This is a solo-maintained open source project. Responses are best-effort and typically happen within a few days to a week, depending on the issue. There are no SLA commitments.

- Bug reports with a clear reproduction: highest priority
- Feature requests: reviewed periodically against the project roadmap (see [ROADMAP.md](ROADMAP.md))
- Security reports: addressed as quickly as possible

If you need guaranteed support turnaround, this preset may not be the right fit for your organization.
